# Contributing

This repo tracks public DAO-ops work.

Before opening an issue, check that the content is public-safe. Do not include secrets, private commercial context, personal data, exploitable security details, signer/control details, or privileged legal/tax material.

Good issues have:

- a precise title;
- a clear outcome;
- relevant public context;
- acceptance criteria or a completion signal;
- links to public references where possible.

Examples:

- Good: `Clarify BR4 approval flow in public governance docs`
- Good: `Fix yETH APY display on mobile viewport`
- Weak: `Governance stuff`
- Weak: `Fix app`

Maintainers may close issues that are too vague, duplicate, obsolete, not actionable, or unsafe for public tracking.
