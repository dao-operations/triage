# Security

Do not disclose exploitable vulnerabilities, secrets, private URLs, key material, signer-control details, or other sensitive operational details in public issues.

For security-sensitive work:

1. Do not open a public issue with actionable exploit details.
2. Use the appropriate private disclosure path for the affected system.
3. If public tracking is needed, create a minimal public-safe placeholder issue without exploit details.
4. Move the detailed remediation plan to a private tracker.

Placeholder format:

```md
Title: Track private remediation for [public-safe component name]

Public note: Details are being handled privately because the issue may be security-sensitive.

Public completion signal: Remediation complete and safe public summary posted.
```
